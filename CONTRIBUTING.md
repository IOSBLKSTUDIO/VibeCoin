# English

Thank you for your interest in contributing to VibeCoin! We welcome contributions from the community and are excited to have you join the VibeCoding movement.

## Code of Conduct

By participating in this project, you agree to maintain a respectful and inclusive environment. We follow the VibeCoding philosophy: **code with feeling, build with passion, create with vibes**.

## How to Contribute

### 1. Fork the Repository

```bash
# Fork via GitHub UI, then clone your fork
git clone https://github.com/YOUR_USERNAME/VibeCoin.git
cd VibeCoin
```

### 2. Create a Branch

Always create a new branch for your work:

```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/your-bug-fix
# or
git checkout -b docs/your-documentation-update
```

**Branch naming conventions:**
- `feature/` - New features
- `fix/` - Bug fixes
- `docs/` - Documentation updates
- `refactor/` - Code refactoring
- `test/` - Adding or updating tests

### 3. Make Your Changes

- Write clean, readable code
- Follow existing code style and conventions
- Add comments where necessary
- Update documentation if needed
- Add tests for new functionality

### 4. Commit Your Changes

Write clear, meaningful commit messages:

```bash
git add .
git commit -m "feat: add wallet balance display"
# or
git commit -m "fix: resolve transaction signing issue"
# or
git commit -m "docs: update README with installation steps"
```

**Commit message prefixes:**
- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation
- `style:` - Code style (formatting, no code change)
- `refactor:` - Code refactoring
- `test:` - Adding tests
- `chore:` - Maintenance tasks

### 5. Push and Create a Pull Request

```bash
git push origin feature/your-feature-name
```

Then go to GitHub and create a Pull Request to the `master` branch.

## Pull Request Requirements

**All contributions MUST go through a Pull Request that will be reviewed before merging.**

Your PR must:

- [ ] Have a clear, descriptive title
- [ ] Include a detailed description of changes
- [ ] Reference any related issues (e.g., "Fixes #123")
- [ ] Pass all existing tests
- [ ] Include new tests if adding functionality
- [ ] Follow the code style guidelines
- [ ] Have no merge conflicts with `master`

### Review Process

1. Submit your Pull Request
2. Maintainers will review your code
3. Address any requested changes
4. Once approved, your PR will be merged

**Note:** PRs may take 1-7 days to review depending on complexity.

## What We're Looking For

### Priority Contributions

- Core blockchain implementation
- Wallet functionality
- Smart contract engine
- Security improvements
- Documentation improvements
- Bug fixes
- Test coverage

### Good First Issues

Look for issues labeled `good first issue` - these are perfect for newcomers!

## Development Setup

```bash
# Clone your fork
git clone https://github.com/YOUR_USERNAME/VibeCoin.git
cd VibeCoin

# Install dependencies
npm install

# Run tests
npm test

# Start local node
npm run node
```

## Questions?

- Open an issue for bugs or feature requests
- Start a discussion for general questions
